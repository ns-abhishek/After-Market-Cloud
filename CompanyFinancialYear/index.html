API Support for Financial Year

This section outlines the API endpoints available for other systems to understand and interact with the financial year configurations.

  • GET /api/financial-years: Retrieve all financial year definitions.
  • GET /api/financial-years/{id}: Retrieve a specific financial year definition by ID.
  • POST /api/financial-years: Create a new financial year definition.
  • PUT /api/financial-years/{id}: Update an existing financial year definition.
  • GET /api/periods/{financialYearId}: Retrieve accounting periods for a given financial year.
  • POST /api/data-migration/import: Endpoint for importing financial data.

(Note: These are conceptual API endpoints for demonstration purposes.)

Data Migration Tools

Tools to facilitate the migration of historical financial data to the new financial year structure during implementation.

Audit Logging

A comprehensive audit log of all financial year configurations, changes, and period closures is maintained. This ensures traceability and compliance. (Refer to the Audit Trails section under Reporting & Analytics for the log display.)

User Role and Permission Management

Access to financial year configuration and period management functionalities is controlled through user roles and permissions. (Refer to the User Management tab for role assignment.)

System Date Independence

The system is designed to be independent of the underlying server's calendar year settings. All financial year logic, including period calculations and status determinations, is driven solely by the configured definitions within the application. This ensures consistency across different geographical deployments and server environments, regardless of local system date settings.
